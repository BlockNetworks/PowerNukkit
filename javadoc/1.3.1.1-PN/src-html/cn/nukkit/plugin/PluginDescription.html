019/**
020 * 描述一个Nukkit插件的类。<br>
021 * Describes a Nukkit plugin.
022 *
023 * 在jar格式的插件中，插件的描述内容可以在plugin.yml中定义。比如这个：<br>
024 * The description of a jar-packed plugin can be defined in the 'plugin.yml' file. For example:
025 * <blockquote><pre>
026 * <b>name:</b> HelloWorldPlugin
027 * <b>main:</b> com.cnblogs.xtypr.helloworldplugin.HelloWorldPlugin
028 * <b>version:</b> "1.0.0"
029 * <b>api:</b> ["1.0.0"]
030 * load: POSTWORLD
031 * author: 粉鞋大妈
032 * description: A simple Hello World plugin for Nukkit
033 * website: http://www.cnblogs.com/xtypr
034 * permissions:
035 *  helloworldplugin.command.helloworld:
036 *   description: Allows to use helloworld command.
037 *   default: true
038 * commands:
039 *  helloworld:
040 *   description: the helloworld command
041 *   usage: "/helloworld"
042 *   permission: helloworldplugin.command.helloworld
043 * depend:
044 * - TestPlugin1
045 * </pre></blockquote>
046 * 在使用plugin.yml来定义插件时，{@code name}、{@code main}、{@code version}、{@code api}这几个字段是必需的，
047 * 要让Nukkit能够正常加载你的插件，必须要合理地填写这几个字段。<br>
048 * When using plugin.yml file to define your plugin, it's REQUIRED to fill these items:
049 * {@code name},{@code main},{@code version} and {@code api}.You are supposed to fill these items to make sure
050 * your plugin can be normally loaded by Nukkit.<br>
051 *
052 * <p>接下来对所有的字段做一些说明，<b>加粗</b>的字段表示必需，<i>斜体</i>表示可选：（来自
053 * <a href="http://www.cnblogs.com/xtypr/p/nukkit_plugin_start_from_0_about_config.html">粉鞋大妈的博客文章</a>）<br>
054 * Here are some instructions for there items, <b>bold</b> means required, <i>italic</i> means optional: (From
055 * <a href="http://www.cnblogs.com/xtypr/p/nukkit_plugin_start_from_0_about_config.html">a blog article of @粉鞋大妈</a>)
056 * </p>
057 * <ul>
058 * <li><b>name</b><br>
059 * 字符串，表示这个插件的名字，名字是区分不同插件的标准之一。
060 * 插件的名字<i>不能包含“nukkit”“minecraft”“mojang”</i>这几个字符串，而且不应该包含空格。<br>
061 * String, the plugin name. Name is one of the ways to distinguish different plugins.
062 * A plugin name <i>can't contain 'nukkit' 'minecraft' 'mojang'</i>, and shouldn't contain spaces.</li>
063 * <li><b>version</b><br>
064 * 字符串，表示这个插件的版本号。使用类似于1.0.0这样的版本号时，应该使用引号包围来防止误识别。<br>
065 * String, the version string of plugin. When using the version string like "1.0.0",
066 * quotation marks are required to add, or there will be an exception.</li>
067 * <li><b>api</b><br>
068 * 字符串序列，表示这个插件支持的Nukkit API版本号列表。插件作者应该调试能支持的API，然后把版本号添加到这个列表。<br>
069 * A set of String, the Nukkit API versions that the plugin supports. Plugin developers should debug in different
070 * Nukkit APIs and try out the versions supported, and add them to this list. </li>
071 * <li><b>main</b><br>
072 * 字符串，表示这个插件的主类。插件的主类<i>不能放在“cn.nukkit”包下</i>。<br>
073 * String, the main class of plugin. The main class<i> can't be placed at 'cn.nukkit' package</i>.</li>
074 * <li><i>author</i> or <i>authors</i><br>
075 * 字符串/字符串序列，两个任选一个，表示这个插件的作者/作者列表。<br>
076 * String or A set of String. One of two is chosen, to describe the author or the list of authors.</li>
077 * <li><i>website</i><br>
078 * 字符串，表示这个插件的网站。插件使用者或者开发者可以访问这个网站来获取插件更多的信息。
079 * 这个网站可以是插件发布帖子或者插件官网等。<br>
080 * String, the website of plugin. More information can be found by visiting this website. The website
081 * can be a forum post or the official website.</li>
082 * <li><i>description</i><br>
083 * 字符串，表示这个插件的一些描述。<br>
084 * String, some description of plugin.</li>
085 * <li><i>depend</i><br>
086 * 序列，表示这个插件所依赖的一个或一些插件的名字的列表。参见：{@link PluginDescription#getDepend()}<br>
087 * List, strings for plugin names, what is depended on by this plugin. See:
088 * {@link PluginDescription#getDepend()}</li>
089 * <li><i>prefix</i><br>
090 * 字符串，表示这个插件的消息头衔。参见：{@link PluginDescription#getPrefix()}<br>
091 * String, the message title of the plugin. See: {@link PluginDescription#getPrefix()}</li>
092 * <li><i>load</i><br>
093 * 字符串，表示这个插件的加载顺序，或者说在什么时候加载。参见：{@link PluginLoadOrder}<br>
094 * String, the load order of plugin, or when the plugin loads. See: {@link PluginLoadOrder}</li>
095 * <li><i>commands</i><br>
096 * 序列，表示这个插件的命令列表。<br>
097 * List, the command list.</li>
098 * <li><i>permissions</i><br>
099 * 序列，表示这个插件的权限组列表。<br>
100 * List, the list of permission groups defined.</li>
101 * </ul>
102 *
103 * @author MagicDroidX(code) @ Nukkit Project
104 * @author iNevet(code and javadoc) @ Nukkit Project
105 * @author 粉鞋大妈(javadoc) @ Nukkit Project
106 * @see Plugin
107 * @see PluginLoadOrder
108 * @since Nukkit 1.0 | Nukkit API 1.0.0
109 */

132    public PluginDescription(String yamlString) {
133        DumperOptions dumperOptions = new DumperOptions();
134        dumperOptions.setDefaultFlowStyle(DumperOptions.FlowStyle.BLOCK);
135        Yaml yaml = new Yaml(dumperOptions);
136        this.loadMap(yaml.loadAs(yamlString, LinkedHashMap.class));
137    }
138
139    private void loadMap(Map<String, Object> plugin) throws PluginException {
140        this.name = ((String) plugin.get("name")).replaceAll("[^A-Za-z0-9 _.-]", "");
141        if (this.name.equals("")) {
142            throw new PluginException("Invalid PluginDescription name");
143        }
144        this.name = this.name.replace(" ", "_");
145        this.version = String.valueOf(plugin.get("version"));
146        this.main = (String) plugin.get("main");
147        Object api = plugin.get("api");
148        if (api instanceof List) {
149            this.api = (List<String>) api;
150        } else {
151            List<String> list = new ArrayList<>();
152            list.add((String) api);
153            this.api = list;
154        }
155        if (this.main.startsWith("cn.nukkit.") && !this.main.equals("cn.nukkit.plugin.PowerNukkitPlugin") && !name.equals("PowerNukkit")) {
156            throw new PluginException("Invalid PluginDescription main, cannot start within the cn.nukkit. package");
157        }
158
159        if (plugin.containsKey("commands") && plugin.get("commands") instanceof Map) {
160            this.commands = (Map<String, Object>) plugin.get("commands");
161        }
162
163        if (plugin.containsKey("depend")) {
164            this.depend = (List<String>) plugin.get("depend");
165        }
166
167        if (plugin.containsKey("softdepend")) {
168            this.softDepend = (List<String>) plugin.get("softdepend");
169        }
170
171        if (plugin.containsKey("loadbefore")) {
172            this.loadBefore = (List<String>) plugin.get("loadbefore");
173        }
174
175        if (plugin.containsKey("website")) {
176            this.website = (String) plugin.get("website");
177        }
178
179        if (plugin.containsKey("description")) {
180            this.description = (String) plugin.get("description");
181        }
182
183        if (plugin.containsKey("prefix")) {
184            this.prefix = (String) plugin.get("prefix");
185        }
186
187        if (plugin.containsKey("load")) {
188            String order = (String) plugin.get("load");
189            try {
190                this.order = PluginLoadOrder.valueOf(order);
191            } catch (Exception e) {
192                throw new PluginException("Invalid PluginDescription load");
193            }
194        }
195
196        if (plugin.containsKey("author")) {
197            this.authors.add((String) plugin.get("author"));
198        }
199
200        if (plugin.containsKey("authors")) {
201            this.authors.addAll((Collection<? extends String>) plugin.get("authors"));
202        }
203
204        if (plugin.containsKey("permissions")) {
205            this.permissions = Permission.loadPermissions((Map<String, Object>) plugin.get("permissions"));
206        }
207    }

211     * Returns the full name of this plugin.
212     *
213     * <p>一个插件完整的名字由{@code 名字+" v"+版本号}组成。比如：<br>

215     * <p>{@code HelloWorld v1.0.0}</p>

279    /**
280     * 返回这个插件所依赖的插件名字。<br>
281     * The names of the plugins what is depended by this plugin.
282     *
283     * Nukkit插件的依赖有这些注意事项：<br>Here are some note for Nukkit plugin depending:
284     * <ul>
285     * <li>一个插件不能依赖自己（否则会报错）。<br>A plugin can not depend on itself (or there will be an exception).</li>
286     * <li>如果一个插件依赖另一个插件，那么必须要安装依赖的插件后才能加载这个插件。<br>
287     * If a plugin relies on another one, the another one must be installed at the same time, or Nukkit
288     * won't load this plugin.</li>
289     * <li>当一个插件所依赖的插件不存在时，Nukkit不会加载这个插件，但是会提醒用户去安装所依赖的插件。<br>
290     * When the required dependency plugin does not exists, Nukkit won't load this plugin, but will tell the
291     * user that this dependency is required.</li>
292     * </ul>
293     *
294     * <p>举个例子，如果A插件依赖于B插件，在没有安装B插件而安装A插件的情况下，Nukkit会阻止A插件的加载。
295     * 只有在安装B插件前安装了它所依赖的A插件，Nukkit才会允许加载B插件。<br>
296     * For example, there is a Plugin A which relies on Plugin B. If you installed A without installing B,
297     * Nukkit won't load A because its dependency B is lost. Only when B is installed, A will be loaded
298     * by Nukkit.</p>
299     *
300     * @return 插件名字列表的 {@code List}对象。<br>A {@code List} object carries the plugin names.
301     * @see PluginDescription
302     * @since Nukkit 1.0 | Nukkit API 1.0.0
303     */
